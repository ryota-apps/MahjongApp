# コードレビュー結果

## レビュー対象
麻雀成績集計アプリ（index.html 単一ファイル PWA）

## 発見事項

### [重要度: 高] renderShopStats が存在しない要素を参照 → ランタイムエラー
- ファイル: index.html:2380
- 問題: `document.getElementById("an-shop")` で参照している `an-shop` 要素がHTML内に存在しない。分析タブを開くたびに TypeError が発生し、店舗別統計が表示されない。
- 修正案: 分析ペイン内に `<div id="an-shop"></div>` を含むカードを追加する。

### [重要度: 高] 保存プレビューと実際の保存値の不一致（場代の扱い）
- ファイル: index.html:1763 vs index.html:1826
- 問題: `updateSaveSummary()` では `net = cash + chipVal` と表示しているが、`saveSession()` では `net = balance + chipVal - venueFee` として保存する。セットで場代を入力した場合、ユーザーに表示される「純収支」と実際に保存される値が異なる。
- 修正案: `updateSaveSummary()` の net 計算に venueFee を反映させる。

### [重要度: 高] 累計純収支チャートが全期間の累計ではなく直近6件のみの合計を表示
- ファイル: index.html:2154-2161
- 問題: `.slice(-6)` した後に running を 0 から加算しているため、全期間の累計ではなく直近6セッションの合計のみが表示される。
- 修正案: 全セッションの累計を計算してから直近6点をプロットする。

### [重要度: 中] JSONインポート時にデータ構造のバリデーションがない
- ファイル: index.html:2625-2687
- 問題: 必須フィールド（date, shop, counts, players）の存在チェックがない。
- 修正案: フィルタリング時に最低限のバリデーションを追加。

### [重要度: 中] セッション編集時に場代（venueFee）を変更できない
- ファイル: index.html:2048-2080
- 問題: 編集パネルに venueFee の編集フィールドがない。

### [重要度: 中] loadSessions() が呼ばれるたびに localStorage の全データをパースする
- ファイル: index.html:1516-1517
- 問題: フィルター変更のたびにフルパースが走る。大量データで遅延の可能性。
- 修正案: メモリキャッシュパターンに変更。

### [重要度: 中] フィルター変更のたびに店舗プルダウンを再構築
- ファイル: index.html:1929-1939, 2112-2122
- 問題: renderHistory/renderAnalysis が呼ばれるたびに shop filter の select を全件リビルド。

### [重要度: 低] CSVエクスポートでヘッダー行がクォートされていない
- ファイル: index.html:2573-2618

### [重要度: 低] Blob URL がメモリリークする
- ファイル: index.html:2546-2551
- 問題: URL.revokeObjectURL() が呼ばれていない。

## まとめ

| 重要度 | 件数 | 概要 |
|--------|------|------|
| 高 | 3 | an-shop要素の欠落、保存プレビューの不一致、累計チャートのロジック誤り |
| 中 | 4 | インポートバリデーション不足、場代編集不可、パフォーマンス関連 |
| 低 | 2 | CSVヘッダークォート、Blob URLリーク |
